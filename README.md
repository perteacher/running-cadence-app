# 러닝 케이던스 측정 · 연습 (MVP)

측면(side-view) 영상에서 러닝 **케이던스(SPM, 분당 걸음수)**를 측정하고, 본인 기준으로 **+5~10%** 목표를 메트로놈으로 연습하게 돕는 단일 파일 웹앱.

- **단일 `index.html`** — 빌드 없음 / 바닐라 JS / CDN만 사용 / GitHub Pages 그대로 배포 가능
- 포즈 인식: **MediaPipe Pose Landmarker**(Tasks Vision, CDN) 우선 → 실패 시 **TF.js MoveNet** 자동 폴백
- 측정 지표는 **케이던스 하나만**. (수직진동·오버스트라이드·전경각 등은 의도적으로 제외 — 추후 확장용으로 포즈 추출부를 모듈로 분리)

## 사용법
1. **환경 선택**: 러닝머신 / 야외.
   - 러닝머신이면 페이스(분/km)를 넣을 수 있음 → 그 속도의 정상 케이던스 대역과 비교해 더 정확히 판정(속도 +1 m/s당 약 +6 SPM 기준).
2. **촬영**: 폰을 삼각대에 **측면·골반 높이**로 세우고 **전신**이 들어오게. **그 자리에서 녹화(12초)** 또는 **영상 업로드** 중 선택. 10~15초면 충분.
3. **결과**: 현재 SPM + 낮음/정상 판정 + 목표 범위(현재 ×1.05~1.10).
4. **연습 모드**: 목표 SPM 메트로놈(소리 + 화면 깜빡임). 한 비트 = 한 걸음.

## 카메라 거치 팁
- **측면**에서, **골반 높이**, 화면에 **머리~발**까지 전신.
- 밝고 배경이 단순한 곳. 펑퍼짐한 옷보다 몸 라인이 보이는 옷이 인식에 유리.
- **러닝머신**: 핸드레일/콘솔이 발목·발을 가리지 않게 레일 **아래로** 비켜 거치.
- **야외**: 러너가 화면 가운데를 지나가게.

## 알려진 한계 (중요)
- **30fps 지터**: 일반 폰 영상은 30fps라 ±1프레임 ≈ **±2~3 SPM** 오차. 결과는 윈도우 평균/반올림으로 표시.
- **원측 다리 가림(occlusion)**: 측면에서는 앞쪽 다리가 뒤쪽 다리를 가려 한쪽 발 인식이 떨어질 수 있음 → 양발 신호를 함께 써서 보완하지만 완전하지는 않음.
- **스케일 무관 지표만**: 케이던스는 *시간* 기반이라 카메라 거리/스케일 보정이 필요 없음(그래서 1순위). 길이 기반 지표(수직진동 cm 등)는 이 MVP에 없음.
- **인식 실패 시**: 전신·측면·밝기를 점검하고 재촬영. 인식 프레임이 30개 미만이면 측정하지 않음.
- 의학적 조언 아님. 케이던스는 폼의 한 지표일 뿐. 통증 지속 시 전문가 상담.

## 안전 원칙 (앱 UI에 반영됨)
- "**180에 맞춰라**" 같은 절대 목표 **금지**. 항상 **본인 현재값 +5~10%** 상대 증가로 안내.
- 결과 화면 고정 경고: *케이던스를 올리면 부하가 사라지는 게 아니라 종아리·아킬레스로 옮겨갑니다. 한 번에 5%씩, 몇 주에 걸쳐 천천히.*

## 배포 (GitHub Pages)
1. 이 폴더(또는 `index.html`)를 리포지토리에 push.
2. Settings → Pages → 브랜치/폴더 지정.
3. `https://<user>.github.io/<repo>/` 접속. 카메라는 **https**에서만 동작(Pages는 https라 OK).

## 기술 메모 (개발자용)
- `index.html` = 배포 단일 파일. 내부에 3개 모듈 블록이 네임스페이스로 분리:
  - `CadenceCore` — 신호처리 + SPM 계산(순수 함수, DOM 없음).
  - `PoseExtract` — MediaPipe/MoveNet 백엔드 추상화. 프레임당 `{leftAnkleY,rightAnkleY,hipMidY,leftVis,rightVis,hipVis,ok}`(정규화 0~1) 반환. **향후 지표 추가 시 이 출력만 확장**하면 됨.
  - `Metronome` + `App` — WebAudio 메트로놈 + UI/통합.
- 알고리즘(연구 문서 §3 그대로): 발목 y 추출 → visibility<0.5 보간 → 저역통과(≈7Hz) → 디트렌드(이동평균 제거) → 발별 피크검출(불응기 fps×0.30, prominence 0.3×std) → 양발 합산 SPM. 신호가 지저분하면 자기상관 지배주파수 폴백. 프레임 시간은 `requestVideoFrameCallback`의 실제 timestamp 사용(가변 fps 대응).
- **회귀 테스트**: `cadence-core.js`는 동일 코어의 독립본 + node 테스트. 실행:
  ```
  node cadence-core.js
  ```
  합성 신호(1.5Hz→180 SPM, 1.4Hz→168, 노이즈/드리프트, 30% 결측)로 4개 케이스 검증. (배포 `index.html`에는 이 코어가 인라인되어 있음.)
